# Code Review — copilotline (`/ai-review all`)

> Generated by the `ai-review` skill: pre-review context agent → 6 specialist
> reviewers (one per lens, `--full` roster) → inline cross-corroboration
> validation. Target = whole repo, real product code = `src/` (3,487 LOC TS),
> `tests/`, `scripts/`, root config, `.github/`. The `.ai-engineering/`,
> `.claude/`, `.codex/` trees are vendored framework tooling and were treated
> as out of scope.

**Product:** `copilotline` — a TypeScript/Bun CLI that renders an ANSI status
line for the GitHub Copilot CLI (reads Copilot session JSON on stdin; fetches &
caches usage quota; manages account selection; installs itself into
`~/.copilot/settings.json`). Hexagonal architecture, zero runtime dependencies,
npm `@arcasilesgroup/copilotline`, Node ≥ 18.

**Severity disputes resolved.** Security scored the SSRF **High** (not Critical)
because exploitation requires influencing the stdin payload, which in the
supported deployment is produced by the Copilot CLI. We keep it as a
**release-blocker** regardless: the project's own `SECURITY.md` classifies stdin
JSON as an untrusted trust boundary and pledges tokens never leave GitHub's API —
this code breaks that pledge, and the fix is one allowlist. Compatibility
correctly declined to double-count the JSONC issue (no prior shipped version to
"break"); correctness owns it as the Critical.

## Top recommendations (ranked — do these, in this order)

1. **[Critical · Security] Allowlist the API host before any token-bearing fetch.**
   `normalizeHost` (`src/infrastructure/copilot-account.ts:282`) strips scheme/slash
   but lets *any* host through, so a payload-controlled `host` sends the user's
   GitHub token to `https://api.<attacker>/…`. Adopt the go-gh model: accept only
   `github.com` and `*.ghe.com` (collapsed to `<last-label>.ghe.com`), fail closed
   otherwise, and validate at the ingestion boundary (account record, `--host`,
   VS Code state) — not just at fetch time.

2. **[Critical · Correctness] Stop mangling the user's `~/.copilot/settings.json`.**
   `applySettingsMutations` (`copilot-settings-file.ts:33`) round-trips through
   `JSON.parse`→`JSON.stringify`, deleting every JSONC comment and trailing comma
   on each `install`/`uninstall`. Apply a surgical textual edit that touches only
   the `statusLine`/`footer.showCustom` keys, or at minimum write a `.bak` and warn.

3. **[Critical · Performance] Make `render` cache-only and resolve the account once.**
   `selectCopilotAccount` runs an eager, un-memoized `gh auth status` + `sqlite3`
   fan-out and is called **3× per render** (`copilot-account.ts:60`;
   `copilot-usage.ts:187,196,206`). Compute the selection once in `runRender`,
   memoize per-process, short-circuit once the payload/config yields a login, and
   push all subprocess detection into the already-detached `refresh` child. This
   also subsumes the **git double-spawn** (`git-info.ts:31`) and the
   **synchronous-blocking render path** (High, performance F2/F3).

4. **[Critical · Testing] Replace false confidence with real coverage + enforce it.**
   Fix the misnamed `configure-status-line.test.ts:28` test to assert the *actual*
   contract, then add unit tests for the untested load-bearing units —
   `value-reader.ts` (the untrusted-input guard used everywhere), `command-tools.ts`,
   `copilot-usage.ts` cache/TTL/debounce, `copilotline-config.ts` malformed-JSON —
   and wire `bun test --coverage` with the 80%/70% floor into CI.

5. **[High · Correctness] Unify the two `quotaFromSnapshot` implementations.**
   The cache path (`copilot-usage.ts:328`) and the payload path
   (`render-status-line.ts:485`) parse the same GitHub shape differently, so an
   *unlimited* (`entitlement: -1`) plan renders `∞` from live data and garbage from
   cache. Extract one shared parser honoring `entitlement === -1` and the
   `quota_remaining` aliases.

6. **[High · Frontend] Honor `NO_COLOR`/`TERM` in the renderer + extract a shared ansi module.**
   `render-status-line.ts` always emits truecolor; only the interactive helper
   checks `NO_COLOR`. Centralize `palette` + `style()` + a `colorEnabled()` resolver
   (kills the duplication and the dead `magenta` entry) and add a truecolor→256→16
   color-depth ladder. Make `NO_COLOR` spec-compliant (disable only when *present
   and non-empty*) and add an explicit `--color`/`--no-color` override.

7. **[High · Correctness] Two quick render-correctness fixes.**
   `formatReset` (`render-status-line.ts:838`) → format in UTC with a `Z`/`UTC`
   label (or inject a TZ) so output is deterministic; `readFlagValue`
   (`cli.ts:676`) → return `undefined` when the next token starts with `-` so
   `render --capture --json` can't write a file named `--json`.

8. **[Medium · Compatibility / Observability] Close the version + diagnostics gaps.**
   Gate the release on `src/version.ts` matching the tag (and interpolate `VERSION`
   into the hardcoded `Editor-Version` headers, `copilot-usage.ts:101`); add
   `doctor` probes for `sqlite3` availability, the `/copilot_internal/user`
   endpoint health, and cache age, so silent degradations become visible.

9. **[Medium · Security] Make SAST actually cover TypeScript.**
   `.semgrep.yml` is Python-only; add TS/JS rules (including a non-literal-`fetch`
   SSRF rule) or document the reliance on CodeQL and add a targeted SSRF query so
   recommendation #1 can't silently regress. Harden the `--capture` write (resolve
   path, `0o600`).

10. **[Medium · Correctness/Perf] Housekeeping.** Atomically create the refresh
    debounce marker to close the TOCTOU (`copilot-usage.ts:207`); distinguish
    empty-vs-malformed stdin in `safeParse` and surface it in `--json`/`doctor`;
    parallelize `enrichAccounts` with `Promise.all` (`cli.ts:413`); delete the dead
    `disableBuiltInFooterItems` path or wire it to a flag.

## What's genuinely good (keep it)

- **Hexagonal layering is clean** — domain imports nothing outward; the boundary
  holds. Don't let new features leak IO into `domain`/`application`.
- **Zero runtime dependencies** and a 44 KB minified single-file bundle → fast cold
  start. Preserve this; prefer the hand-rolled `value-reader` over pulling `zod`.
- **No shell injection** (all 6 spawn sites use arg arrays), **TLS never disabled**,
  **atomic 0o600 writes**, **tokens never persisted to cache**, **untrusted strings
  sanitized** before render (`sanitizeText`).
- **Defensive multi-path field extraction** makes the renderer resilient to Copilot
  JSON schema drift.
- **Real cross-OS CI** (ubuntu/macOS/windows) with a render smoke test, npm
  provenance, and checksum-verified release binaries.
